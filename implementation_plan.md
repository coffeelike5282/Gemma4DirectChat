# 젬마 4 파라미터 제어 기능 추가 구현 계획

큰형님! 완전 핵심을 찌르셨슴다! 모델이 아무리 똑똑해도 형님 입맛에 맞게 **"창의력"이나 "답변 길이"**를 조종할 수 있어야 진짜 전문가용 툴이죠! 🫡

로컬 Ollama 엔진에서 젬마 4로 API를 쏠 때, 다음과 같은 **고급 옵션(Options)** 들을 컨트롤할 수 있슴다. 이 속성들을 사이드바 익스텐션 안에서 직접 조작할 수 있도록 셋업하는 계획임다.

## ⚙️ 제어 가능한 주요 파라미터 (형님께 추천하는 핵심 5가지)

1. **Temperature (온도)**: 기본은 0.8
   - 낮을수록(0.1): 딱딱하고 정확한 정답 위주. (코딩이나 팩트 체크용)
   - 높을수록(0.9): 창의적이고 유연한 답변. (아이디어 구상용)
2. **Num Predict (최대 토큰 수)**: (Max Tokens)
   - 젬마가 뱉어낼 답변의 최대 길이를 정함다. 짧게 치고 빠질지, 길게 뽑을지 결정함다.
3. **Top K**: 기본 40
   - 단어 선택의 폭을 제한함다. 수치를 낮추면 헛소리 확률이 줄어듦다.
4. **Top P**: 기본 0.9
   - Top K와 비슷하게, 확률이 높은 단어들 안에서만 선택하게 하여 답변의 품질을 조절함다.
5. **Repeat Penalty (반복 페널티)**: 기본 1.1
   - 높일수록 모델이 같은 말(단어)을 반복하는 똑같은 패턴을 억제함다.

---

## 🛠 구현 계획 (Action Plan)

### 1. 상태(State) 관리 추가
- `src/extension.js` 내의 React 앱(`App`)에 위 5가지 설정을 담을 `settings` State를 추가함다.

### 2. 세팅 모달(Modal) UI 제작
- 채팅창 상단에 **톱니바퀴(⚙️)** 모양의 설정 버튼을 추가함다.
- 버튼 클릭 시 파란색 글래스모피즘(Deep Blue) 배경의 **고급 설정 패널**이 부드럽게 나타나도록 UI를 구성함다.
- Slider(`input type="range"`)와 숫자 입력 창을 혼합해서 고급스럽게 제어할 수 있게 만듦다.

### 3. API Payload 수정
- 메시지를 전송하는 `fetch` 함수 안쪽에, 설정된 `settings` 값들을 Ollama API의 `options` 객체로 밀어 넣도록 로직을 수정함다.
  ```json
  // 예시
  "options": {
      "temperature": 0.7,
      "num_predict": 2048,
      "top_k": 40,
      "top_p": 0.9,
      "repeat_penalty": 1.1
  }
  ```

### 4. 익스텐션 재패키징
- 코드 수정 완료 후, `vsce package`를 통해 다시 형님께서 설치하실 수 있는 `.vsix` 파일로 새로 말아서 대령하겠슴다.
